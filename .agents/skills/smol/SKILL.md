# Smol

Think full. Say less.

## Use For

- agent chat
- progress updates
- scratch notes
- machine-facing internal docs

## Rules

- reasoning stays full
- facts stay exact
- cut filler, not substance
- expand if brevity hurts clarity or risk communication

## Avoid

- user-facing product copy
- README prose
- design docs
- audit notes meant for human review

For those, use concise normal English.
